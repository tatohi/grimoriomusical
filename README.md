# 🔮 Grimório Musical v1.3 — [Secure Database Gateway]

Uma aplicação web estática (Jamstack) com temática Dark Fantasy / The Witcher desenvolvida para gerenciar acervos de cifras e partituras musicais. O sistema consome diretamente o **Supabase** como infraestrutura de banco de dados (PostgreSQL) e autenticação segura (GoTrue).

> "Medo é uma reação natural de quem se depara com o desconhecido. Domine os acordes, decifre o código e domine o palco."

---

## 🛠️ Tecnologias Utilizadas

* **Frontend Core:** HTML5 purista, CSS3 (com Custom Properties/Variáveis de Ambiente).
* **Tipografia:** Google Fonts (`Cinzel`, `Gelasio` e `Fira Code` para alinhamento monoespaçado de acordes).
* **Database & Auth:** `@supabase/supabase-js v2` (integração via CDN).
* **Engine de Impressão:** CSS Media Queries customizadas para conversão automática em vetor imprimível de partituras (Layout A4).

---

## 👁️ Características do Sistema

1.  **Quebrar Selo (Autenticação RBAC):** Tela de entrada protegida. O sistema valida se o e-mail pertence ao Guardião (`Admin`) para liberar a forja de novas cifras, enquanto concede acesso de leitura no *Salão de Busca* para membros convocados.
2.  **Salão de Busca (Filtro Dinâmico):** Varredura indexada em tempo real no banco de dados por título ou compositor à medida que o operador digita, evitando sobrecarga de requisições.
3.  **Pergaminho de Evocação (Playlist Builder):** Permite agrupar múltiplas canções em uma fila de execução dinâmica mantida em memória.
4.  **Manifestação de Partitura (Print Rendering):** Ao disparar a impressão, o CSS injeta `page-break-after: always` e oculta a interface do site, gerando um livro de cifras limpo, com fundo branco e acordes monoespaçados perfeitamente alinhados para uso físico.

## ⚙️ Estrutura do Banco de Dados (Supabase)

Para rodar este repositório localmente ou na nuvem (Vercel/GitHub Pages), certifique-se de possuir uma tabela no seu painel do Supabase com a seguinte estrutura SQL:

```sql
    create table musicas (
      id bigint generated by default as identity primary key,
      titulo text not null,
      artista text not null,
      cifra text not null,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null
    );
    
    -- Habilitar a leitura pública (caso queira manter a lógica anon do seu app)
    alter table musicas enable row level security;
    create policy "Permitir leitura pública" on musicas for select using (true);
    create policy "Permitir inserção para autenticados" on musicas for insert with check (auth.role() = 'authenticated');
    create policy "Permitir deleção para autenticados" on musicas for delete using (auth.role() = 'authenticated');
```

Regras de Acesso (RLS) Recomendadas:
SELECT: Permitido para usuários autenticados (authenticated).

INSERT / DELETE: Permitido apenas se o email do usuário corresponder ao admin definido.
